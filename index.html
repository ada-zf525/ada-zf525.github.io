The ESEGraph library

This library is hosted at https://github.com/ese-msc/esegraph and is included as a git submodule in this repository. Submodules are a way to include one git repository inside another, while allowing each to be managed independently. You can find the code for the ESEGraph library in the esegraph directory of this repository.

To clone this repository along with the ESEGraph submodule, you can use the following command:

git clone --recursive <repository_url>

To update the ESEGraph library to the latest version (if you are told to), you can use the following commands in your terminal:

git submodule update --remote

If your group discovers a bug in the ESEGraph library, you can report it by opening an issue on the ESEGraph GitHub repository. If you have a fix for the bug, you can fork the ESEGraph repository, make your changes, and then submit a pull request to the original repository.

Mass-balance simulator

The mass-balance simulator evaluates an encoded circuit vector and returns the economic value used by the genetic algorithm. It is the deterministic forward model for the project: a proposed circuit vector is converted into steady-state product stream compositions, and those product streams are converted into a scalar fitness value.

The GA-facing interface is:

double circuit_performance(std::span<const int> circuit_span);

The diagnostic result reports:

  • whether the vector shape was valid;
  • whether the fixed-point mass-balance iteration converged;
  • the number of iterations used;
  • final palusznium product, gormanium product, and tailings stream compositions;
  • the mass-conservation residual;
  • the economic value;
  • a failure reason, where applicable.

Build

cmake -B build -S .
cmake --build build -j

Run tests

ctest --test-dir build --output-on-failure

Evaluate a sample circuit

./build/tools/bin/evaluate_circuit --sample simple_a
./build/tools/bin/evaluate_circuit --sample simple_b
./build/tools/bin/evaluate_circuit --sample recycle3
./build/tools/bin/evaluate_circuit --sample recycle4

Evaluate a custom circuit vector

Circuit vectors may be comma-separated or whitespace-separated:

./build/tools/bin/evaluate_circuit "1,3,3,2,2,2,0,1,2,3,0,4,5"

Run the mass-balance benchmark

./build/benchmarks/bin/benchmark_mass_balance 10000

or use the helper script:

./scripts/run_mass_balance_benchmark.sh 10000

Benchmark output columns are:

case,repetitions,total_time_s,mean_time_us,converged,iterations,value_per_s,mass_residual_kg_s

No-recycle cases mainly measure unit-equation overhead. Recycle cases stress the fixed-point convergence behaviour. A solver change should only be treated as an improvement if convergence, mass residual, and economic value remain consistent.

Current diagnostic workflow

A useful local workflow when changing the mass-balance implementation is:

cmake -B build -S .
cmake --build build -j
ctest --test-dir build --output-on-failure
./scripts/evaluate_sample_circuits.sh
./scripts/run_mass_balance_benchmark.sh 10000

This checks correctness-oriented invariants first, then produces timing data for the fixed benchmark circuits.
