Palusznium Rush

Overview

Palusznium Rush is a C++20 project that optimises mineral-processing circuits with a genetic algorithm. The codebase consists of three components:

  • A mass-balance simulator that converts an encoded circuit vector into steady-state product streams and a scalar economic value.
  • A genetic algorithm that searches over circuit vectors using the simulator as a fitness function.
  • A non-interactive command-line interface that drives both components from shell scripts and HPC job systems.

Mass-Balance Simulator:

  • CSimulator runs a fixed-point iteration over the circuit graph. Each unit splits its inlet stream into concentrate(s) and tailings using component-specific constants, such that Type A: 2 outlets and Type B: 3 outlets. The iteration repeats until stream values converge, and then the economic model is applied to the final product streams to return a scalar fitness value.
  • Our unique design choice includes accounting for non-convergence behaviour. If the fixed-point loop exceeds options_.max_iterations without the maximum relative change falling below options_.relative_tolerance, the result is flagged converged = false, a failure_reason string is populated, and the configured failure_fitness value is returned instead of an economic score.

Genetic Algorithm (GA):

  • Genetic_Algorithm implements a steady state GA that evolves circuit vectors to maximise economic fitness. A population of candidates is initialised (with up to 100× attempts to seed valid circuits), then evolved via tournament selection, single-point crossover, and per-gene mutation. Both a fixed mode (unit types locked in the prefix) and a swappable mode (unit types co-evolved alongside routing) are supported in our implementation.

Building

git clone --recursive https://github.com/ese-ada-lovelace-2025/acds-palusznium-rush-acanthite.git
cmake -B build -S .
cmake --build build -j

The main executable is produced at build/bin/Circuit_Optimizer.

Command-Line Interface

Circuit_Optimizer parses its arguments with the cli::CliParser class and dispatches to one of four operation modes through cli::CommandExecutor. Comma-separated values (CSV) are written to standard output by default, and progress as well as error messages are written to standard error so that the two streams remain separated.

For further detail on Command-Line Interface (CLI) commands, usage, operation modes, and examples, please refer to docs/command_line_interface.md.

Development Guide

Our project includes a test suite, a diagnostic tool for inspecting individual circuits, a mass-balance benchmarker for measuring solver performance, and Doxygen-generated API documentation.

For setup instructions and usage examples for all of the above, see docs/DEVELOPMENT.md.

ESEGraph Submodule

The ESEGraph library is hosted at https://github.com/ese-msc/esegraph and is included as a git submodule under esegraph/. To update the submodule to the latest revision:

git submodule update --remote

Bug reports and fixes for the ESEGraph library should be submitted to the upstream repository.
