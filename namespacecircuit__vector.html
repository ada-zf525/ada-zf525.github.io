◆ destination_kind()

DestinationKind circuit_vector::destination_kind ( int destination,
int n_units )

Classifies an output destination integer relative to a circuit of n_units units.

The circuit encoding indicates and reserves destinations as shown below:

  • n_units: Palusznium product outlet (DestinationKind::Palusz)
  • n_units + 1: Gormanium product outlet (DestinationKind::Gorman)
  • n_units + 2: Tailings outlet (DestinationKind::Tailings)
  • otherwise: DestinationKind::Invalid
Parameters
destinationRaw destination value read from a circuit vector.
n_unitsTotal numer of separation units in the circuit (which must be larger than zero).
Returns
DestinationKind corresponding to where the destination maps to.

◆ parse_circuit_vector()

ParseResult circuit_vector::parse_circuit_vector ( std::span< const int > circuit_vector)

Parses a flat integer circuit vector into a structured ParseResult.

There are several constraints enforced, such as:

  • exactly one feed input (n_inputs == 1).
  • at least one unit (n_units > 0).
  • exactly three product streams (n_product_streams == 3).
  • each unit has either 2 (Type A) or 3 (Type B) outputs.
  • the feed must route to a unit, and not directly to an outlet.
  • all destination values must be in the valid range between 0 and n_units + 2.
  • the total vector lenght must match exactly with the declared structure.
Parameters
circuit_vectorThe flat integer encoding of the circuit.
Returns
A ParseResult describing the outcome of the parse.
