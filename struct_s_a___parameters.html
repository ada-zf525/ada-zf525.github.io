Detailed Description

Tuning parameters for the simulated-annealing optimiser.

Each field controls one aspect of the annealing schedule, neighbour generation policy, or termination behaviour. The default values are conservative and suitable for the benchmark cases shipped with this project. callers should tune them per problem.

int SA_Parameters::initial_attempt_multiplier = 1000

Lower bound on the number of attempts used to seed an initial valid state.

The effective number of attempts is the maximum of one hundred and this value. If no valid state is found within the budget the optimiser returns a failure marker. The same budget is reused per restart.

int SA_Parameters::restart_count = 0

Maximum number of random restarts permitted once max_stagnation is reached.

On a restart the trajectory state is reseeded from a fresh random candidate while the global best is preserved. A value of zero disables restarts. once the budget is exhausted the optimiser terminates.

double SA_Parameters::type_mutation_probability = 0.02

Probability of flipping one unit's type in a swappable-case neighbour.

Only consulted by optimize_swappable_span_sa. With this probability a random unit's type is toggled between Type A and Type B and a fresh wiring vector is generated for the new prefix. otherwise the existing wiring is mutated in place. Values are clamped to the closed interval [0, 1].
